# Project Context — Transformers
Generated by Transformers init on 2026-03-18

## Overview
Transformers is a Claude Code plugin providing multi-agent orchestration for software development. It ships 14 specialized agents (7 Autobots for building, 5 Decepticons for testing, 2 orchestrators) coordinated by two orchestrators — Optimus for build workflows, Megatron for test campaigns. Users install it from the Claude Code marketplace and interact via slash commands and conversational directives.

## Tech Stack
- **Language/format:** Markdown with YAML frontmatter (no compiled code)
- **Runtime:** Claude Code plugin system
- **Package/distribution:** Claude Code marketplace (`/plugin marketplace add ShankarKakumani/transformers`)
- **Current version:** 3.4.1 (defined in both `.claude-plugin/marketplace.json` and `plugins/transformers/.claude-plugin/plugin.json` — must stay in sync)
- **CI:** GitHub Actions (`.github/workflows/bump-version.yml`)

## Command Types

### Lifecycle Commands (5 phases, 2 human approval gates)
- `feature` — gather → research → plan → build → review → test → summary
- `bugfix` — gather → investigate → plan → fix → verify → summary

Artifacts in `.claude/transformers/.temp/{type}/{name}/`:
- `00-gather.md`, `01-research.md`, `02-plan.md`, `03-build-log.md`, `04-review.md`
- `status.md` — compaction recovery checkpoint
- `05-tokens.md` — per-agent token usage tracking

### Standalone Commands
`brainstorm`, `research`, `explain`, `refactor`, `debug`, `test`, `pr-generator`, `commit-generator`, `commit-and-push`, `init`, `report`

## Key Patterns

### Agent Definition Format
YAML frontmatter fields: `name`, `description`, `model`, `tools` (list), `permissionMode` (optional, `acceptEdits` for builders), `maxTurns`, `background` (optional), `memory` (optional, `project` for orchestrators), `skills` (list)

Orchestrators have `Agent` tool — builders/testers do not. Prowl has no `Write`/`Edit` tools.

### Skill Bundle Format
Each skill: `skills/{name}/SKILL.md` + optional content files. SKILL.md frontmatter: `name`, `description`, `user-invocable: true|false`. Content referenced via `@${CLAUDE_PLUGIN_ROOT}/skills/{name}/file.md`.

### Sub-agent Result Discipline
Critical: orchestrators instruct every sub-agent to write detail to artifact files and return only 1-3 line summaries. Prevents context window exhaustion.

### auto-init Middleware
Loaded by Optimus and Megatron on every command. Handles: project context staleness checks, memory hydration (long-term index + temp), artifact TTL pruning (2-day TTL on `.temp/` subdirs), in-progress work detection, activity logging.

### Compaction Recovery
Every lifecycle phase updates `status.md` with `phase`, `status`, `summary`, `next`. After `/compact`, first action is always read `status.md` then relevant artifact files.

### Memory System
Two tiers: `memory/temp.md` (per-conversation) and `memory/long-term/` (per-project, cross-session). Only orchestrators graduate temp → long-term at lifecycle end. Sub-agents propose via `[MEMORY]`/`[LONG-TERM]` tags.

### HARD RULES Pattern
Every command leads with an unnegotiable HARD RULES block. Universal rules: never `subagent_type: general-purpose/Explore/Plan`, never Co-Authored-By, never commit without showing the message.

## Git Workflow
- **Strategy:** Trunk-based, direct pushes to `main`, no feature branches
- **Commit style:** Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`)
- **Tag pattern:** `vMAJOR.MINOR.PATCH` (current: `v3.4.1`)
- **Release flow:** Push tag → CI extracts version → updates both version files → commits `chore: bump version to X.Y.Z` → pushes to `main` → creates GitHub Release
- **Version files:** Both `.claude-plugin/marketplace.json` and `plugins/transformers/.claude-plugin/plugin.json` — CI keeps them in sync via `jq`
- **No pre-commit hooks** — commits are instant
- **Gitignore:** Only `.claude/transformers/.temp/` is excluded

## Gotchas
1. **CI writes back to `main` after every tag push** — always `git pull` after pushing a tag before doing more work
2. **Version must stay in sync** in both `marketplace.json` and `plugin.json` — CI handles this on tag push, but manual edits must update both
3. **hooks.json has a SessionStart hook** — CLAUDE.md claims it's intentionally empty, but it's not; CLAUDE.md is stale on this point
4. **Orchestrators cannot write code** — no `Write`, `Edit`, or `Bash` tools; if you add these, you break the architecture
5. **Prowl is strictly read-only** — never give it `Write`/`Edit` tools
6. **`auto-init` is the system's middleware** — it runs on every orchestrator command; changes here affect every workflow
7. **Token tracking is built-in** — `05-tokens.md` per run with self-improvement loop; the system optimizes its own future token usage
8. **Autonomous mode** — lifecycle gate skipping is valid when called programmatically; still checkpoint to files and escalate after 2 failed auto-fix attempts
