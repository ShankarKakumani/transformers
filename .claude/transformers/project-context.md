## Overview
Transformers is a Claude Code plugin that provides a team of AI-powered agents for build and test orchestration. Autobots (led by Optimus) build features collaboratively, and Decepticons (led by Megatron) attack code with tests. Each agent has a distinct personality and thinking style that shapes its approach.

## Tech Stack
- **Language:** Markdown-based plugin definitions (no compiled code)
- **Platform:** Claude Code plugin system
- **Version:** 2.5.0
- **Author:** ShankarKakumani
- **CI/CD:** GitHub Actions — auto-bumps version on `v*` tag push and creates release

## Architecture
This is a Claude Code plugin, not a traditional application. All "code" is prompt engineering in markdown files.

```
plugins/transformers/
├── .claude-plugin/plugin.json    — Plugin metadata and version
├── agents/                       — 13 agent definitions (personality + tools + rules)
├── commands/                     — 12 slash commands (user entry points)
├── skills/                       — 5 internal skills (inject knowledge into agents)
├── hooks/hooks.json              — Empty (security handled by core-principles)
└── knowledge/                    — 4 knowledge bases (shared rules and patterns)
```

### Layer Map
- **Commands** = user-facing entry points (`/transformers:feature`, `/transformers:test`, etc.)
- **Agents** = the workers that commands spawn (Optimus, Bumblebee, Megatron, etc.)
- **Skills** = knowledge injected into agents at spawn time (thin wrappers pointing to knowledge files)
- **Knowledge** = authoritative source of patterns, principles, checklists

## Knowledge System
Four knowledge files provide shared rules:
1. **core-principles.md** — Universal rules for all agents (honesty, impact analysis, security, scope discipline)
2. **build-patterns.md** — Decomposition strategy, agent selection matrix, conflict resolution, quality gates
3. **test-strategies.md** — Coverage matrix, attack patterns per Decepticon, reporting format, stop criteria
4. **code-review-checklist.md** — Prowl's rulebook: pattern consistency, architecture, code quality, safety

Skills are thin wrappers that inject knowledge into agents via `@${CLAUDE_PLUGIN_ROOT}/knowledge/...` includes.

Exception: `auto-init` skill has inline logic — loads project context, checks for in-progress artifacts, and delegates activity logging to Scribe. Loaded by Optimus and Megatron on every command.

## Key Patterns
- **Orchestrators never write code** — Optimus and Megatron only plan, delegate, and report
- **Prowl never writes code** — review only, delivers verdicts (Approved / Changes Requested / Blocked)
- **All Decepticons are read-only** — they test and report, never fix
- **Parallel execution** — independent tasks run via multiple Agent tool calls in one message
- **Human approval gates** — architectural decisions, scope changes, irreversible actions always confirmed
- **Severity system** — MUST FIX / SHOULD FIX / NITPICK (reviews) and CRITICAL / HIGH / MEDIUM / LOW (tests)
- **Background agents** — Soundwave, Shockwave, Starscream, Barricade run in background mode
- **Scribe for persistence** — Optimus/Megatron can't write files, so they spawn Scribe to log activity and write reports
- **Artifact tracking** — `bugfix` and `feature` commands write phase artifacts to `.claude/transformers/active/` for context recovery after compaction
- **No co-author attribution** — Never add Co-Authored-By to commits or PRs

## Gotchas
- **No source code** — This is entirely markdown-based prompt engineering. "Bugs" are prompt issues, not code issues.
- **Hooks are intentionally empty** — Security is handled by core-principles loaded into every agent, not by plugin hooks (which fire globally and affect other plugins).
- **Skills are thin wrappers** — Don't edit skills directly; edit the knowledge files they point to.
- **auto-init is different** — Unlike other skills, its logic is inline in SKILL.md, not in a separate knowledge file.
- **Version bumping is automated** — Push a `v*` tag and the GitHub Action updates both JSON files and creates a release.
- **settings.local.json has stale entries** — Contains permission grants from dev/testing including references to removed `detect.sh` scripts.
- **Scribe only writes to `.claude/transformers/`** — Never touches source code. Only handles logs, reports, and memory files.
- **PR preferences are remembered** — `pr-generator` saves account/branch/reviewer preferences to `.claude/transformers/pr-preferences.md` for next time.
- **`estimate` and `ship` were removed** — Replaced by `commit-generator` and `pr-generator` respectively. `bugfix` is new.
